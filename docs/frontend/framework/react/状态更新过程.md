# 状态更新过程


## 状态更新的“三部曲”

当我们调用 `setCount(prev => prev + 1)` 时，React 内部经历了以下过程：

**第一步：创建 Update 对象**

React 会创建一个 Update 对象，记录本次更新的动作（比如是一个具体的值，还是一个函数）。

**第二步：入队与调度**

这个 Update 会被推入该 Hook 自身的 queue 中。同时，React 会触发一个“调度”任务。它并不会立即重新渲染组件，而是通知调度器（Scheduler）：“这个组件有变化，请在空闲时安排一次演习。”

**第三步：Render 阶段的“状态合并”**

当 React 决定重新渲染该组件时：

- 重新执行函数组件。

- 再次运行到 `useState` 时，React 会遍历 `queue` 里的所有 Update。

- 根据上一次的状态，依次执行这些 Update，最终计算出最新的 `memoizedState`。
