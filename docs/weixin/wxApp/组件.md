# 组件


## 数据监听器

微信小程序的监听器定义在 `observers` 对象中，类似于 Vue 的 `watch`，某些场景下，比 `watch` 更灵活。

- 监听整个对象。

    可以使用 `**` 通配符监听对象的所有子数据的变化。

    ```javascript
        Page({
            data: {
                user: {
                    name: '',
                    detail: {
                        province: ''，
                        city: '',
                    }
                }
            }
            observers： {
                // 触发条件：
                // #1 使用 setData 设置 this.data.user.detail 本身或其下任何
                //    子数据字段时触发
                // #2 使用 setData 设置 this.data.user 也会触发
                'user.detail.**': (newVal) => {
                    // 回调参数：
                    // newVal 接收到的是整个 user.detail 对象的最新值，而不是
                    //        变化的那个字段。
                    console.log('数据变化'， newVal)
                } 
            }
            attached: () => {
                // 这样会触发上面的 observer
                this.setData({
                    'user.detail': { /* ... */ }
                });

                // 这样会触发上面的 observer
                this.setData({
                    'user.detail.city': 'xxx'
                });

                // 这样也会触发上面的 observer
                this.setData{
                    user: {/* ... */}
                }
            }
        })
        ```
- 监听多个字段或属性

    可以用逗号 `,` 分隔，同时监听多个属性。

    ```javascript
    Page({
        data: {
            a: 0, 
            b: 0,
            sum: 0,

            user: {
                name: '张三',
                age: 19,
                detail: {
                    province: '陕西',
                    city: '西安'
                }
            }
        },
        observers: {
            // #obs1
            'a, b': (a, b) => {
                this.setData({
                    sum: a + b
                })
            },
            // #obs2
            'user.name, user.age': (newName, newAge) => {
                console.log('名字或年龄变了', newName, newAge);
            }
        },
        attached: () => {

            // 触发obs1
            this.setData: {{
                'a': 3
            }};

            // 触发obs1
            this.setData: {{
                'b': 4
            }};

            // 触发obs2
            this.setData({
                'user.name': 'Blob'
            });

        }
    })
    ```
- 监听数组

    某些场景下，我们可能需要监听数组的变化. 针对数组的监听，有几种场景：

    - 监听整个数组的“替换”。
    
        监听的数组是数组本身（即引用）的变化，内部某一项属性的变化是不会被监听到的。
        可以监听到数组长度的变化。

    - 监听数组及内部属性的变化。

        和对象的监听一样，使用 `**` 通配符去监听。但只能监听到数组“内部元素”的变化，很难监听数组本身的变化，如“长度”。

    - 监听数组的某一项。
        
        比如你只关心数组里第 0 个元素变没变。

    - 监听数组内对象的属性。

    ```javascript
    Page({
        data: {
            // 简单数组
            tags: ['前端', '后端'],

            // 对象数组
            list: [
                { name: 'Apple', price: 5 },
                { name: 'Banana', price: 3 }
            ]
        },
        observers: {
            // #obs1 监听整个 list 数组的替换
            'list': function(newList) {
                console.log('可以监听到数组长度变化');
                console.log('数组整体被替换了:', newList);
            },
            // #obs2 监听 list 数组及内部属性的变化
            'list.**': function(newList) {
                console.log('无法监听到数组长度变化');
                console.log('数组内部属性发生变化',  newList)
            },
            // #obs3 监听 list 数组的第 0 个元素有没有变化
            'list[0]': function(newItem) {
                console.log('第一个元素变化了:', newItem);
            },
            // #obs4 监听第 0 个用户的 name 属性
            'list[0].name': function(newName) {
                console.log('第一个用户改名了:', newName);
            }
        },
        attached() {
            const newList = [ ...this.data.list, { name: 'Banana', price: 3 } ];

            this.setData({
                list: newList
            })

            this.setData({
                list: this.data.list
            })

            this.setData({
                'list[0]': {
                    name: 'Orange',
                    price: 10
                }
            })

            this.setData({
                'list[0].name': 'Orange'
            })
        }
    })    
    ```

## 生命周期

小程序自定义组件有 5 个生命周期，按照下面流程图中的顺序调用。


![微信小程序组件生命周期](../assets/images/wxapp-component-lifecycle.svg)

其中最重要的是 `created`、`attached` 和 `detached` 这 3 个生命周期。

- `created()`：组件实例创建

    |特性          |说明                          |
    |--------------|-----------------------------|
    |触发时机       |组件实例刚刚被创建好时        |
    |能否 `setData` |❌ 不能                     |
    |`data` 状态    |还是构造器中定义的初始值      |
    |典型用途       |给 `this` 添加自定义属性字段  |
    |调用次数       |组件创建时一次                |

- `attached()`：进入节点树（最重要）

    |特性          |说明                                 |
    |--------------|------------------------------------|
    |触发时机       |组件完全初始化完毕、进入页面节点树后  |
    |能否 `setData` |✅ 可以                            |
    |典型用途       |绝大多数初始化工作的位置：发请求、设状态、查 DOM  |

- `ready()`：渲染层初始化完成

    |特性          |说明                                 |
    |--------------|------------------------------------|
    |触发时机       |当组件被插入页面节点树时         |
    |推荐程度       |大多数情况用 `attached` 代替     |
    |典型用途       |需要确认渲染完成后才能操作的逻辑  |

- `moved()`：组件在节点树中被移动

    |特性          |说明                                 |
    |--------------|------------------------------------|
    |触发时机       |组件实例被移动到节点树的另一个位置时 |
    |注意           |组件没有被销毁，只是位置变了        |

- `detached()`：从节点树移除（销毁）

    |特性          |说明                                 |
    |--------------|------------------------------------|
    |触发时机       |组件离开页面节点树后 |
    |典型用途           |必须清理所有资源：定时器、请求、连接、监听器 |
