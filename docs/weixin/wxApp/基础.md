# 基础


小程序的技术框架深度借鉴了 Vue 的模板语法与 React 的组件化思想。

## 代码构成

在小程序官网中，代码构成包含 WXML、 WXSS、 JS 和  JSON 文件，考虑到 JSON 只是做配置用，这里我们先不管它。

虽然它们看起来和网页开发的 HTML、CSS、JS 很像，但在微信的“双线程架构”下，它们的底层逻辑和运行机制其实发生了根本性的变化。并且为了适配“微信安全管控”，在语法层面做了“增强”和“限制”。

## WXML

小程序官网的[组件](https://developers.weixin.qq.com/miniprogram/dev/component/)列出了全部可用标签。

| HTML 标签 | WXML 对应组件| 说明 |
|-----------|-------------------------| ----- |
|`<div>`    |`<view>`                 | 块级容器 |
| `<span>`  |`<text>`                 |行内文本（只有 `<text>` 支持长按选中）|
| `<img>`   |`<image>`                |图片（必须指定宽高，支持懒加载）|
|`<a>`      |`<navigator>`            |页面跳转（不能跳外链）|
|`<ul>` / `<li>` | `<view>` + `wx:for`| 列表（无原生列表标签）|
|`<input>`  | `<input>` / `<textarea>`|	表单（样式受限，需用 placeholder-style）|


### 数据绑定

使用“双花括号”将 JS 中的数据动态渲染到视图上。

- 基础绑定

    ```xml
    <view>{{ message }}</view>
    <image src="{{ imageUrl}}"><image>
    ```

    ```js
    Page({
        data: {
            message: 'Hello World'
            imageUrl: '/images/logo.png'
        }
    })
    ```

- 支持简单运算， 如三元运算、加减乘除、对象或者数组的访问。

    ```xml
    <view>
        <text>{{ a + b }}</text>
        <text>{{ user.name }}</text>
        <text>{{ list[0].name }}</text>
    </view>
    ```

    ```js
    Page({
        data: {
            a: 2, 
            b: 3,
            user: {
                name: '张三'
            },
            list: [
                {
                    name: 'Apple',
                    age: 13
                }
            ]
        }
    })
    ```


### 条件渲染


条件渲染的原理和 Vue 一样，有两种方式：

- `wx:if` / `wx:elif` / `wx:else`

    这种方式操作的是真实的 DOM，因此性能开销比较大。 
    
    ```xml
    <view wx:if="{{ view == 'WEBVIEW' }}">webview</view>
    <view wx:elif="{{ view == 'APP' }}">App</view>
    <view wx:elif="{{ view == 'MINA'}}">mina</view>
    ```

    类似 Vue 的 `template` , 小程序也有包装元素，不会被渲染为真实的 DOM。可以一次性控制多个元素的显示/隐藏。


    ```xml
    <block vx:if={{ true }}>
        <view>view1</view>
        <view>view2</view>
        <view>view3</view>
    </block>
    ```

- `hidden`

    这种方式控制的是 CSS 的 `display` 属性。

    ```xml
    <view hidden="{{ true }}">view1</view>
    ```

具体表现差异：

|特性 |wx:if | hidden |
|---------------|----------------------------------|-------------------------------------|
|原理           | 条件为 `false` 时， 不渲染 DOM    | 始终渲染，仅通过 CSS 控制显示/隐藏    |
|切换性能消耗   | 高（需要销毁/重建DOM）            | 低（只改CSS）                         |


### 列表渲染

用于循环渲染数组，类似于 Vue 的 `v-for`。

- 默认使用 `item` 和 `index` 作为当前项和索引的变量名。
- 可以使用 `wx:for-item` 和 `wx:for-index` 来指定当前项和索引。
- 可以使用 `wx:key` 为当前项指定 `key`。用于优化列表的渲染性能，确保列表项的顺序变化时能高效更新。
    - 如果是数字或者字符串数组，可以使用 `this` 关键字代表 `item` 本身; 
    - 如果是纯静态列表可以使用 `index`；
    - 其他情况则必须指定列表项中的属性，属性值唯一且为字符串或数字值

```xml
<view>
    <!-- 默认使用 item 作为当前项， index 作为索引 -->
    <view wx:for="list">{{ index }} . {{ item.title }}</view>

    <!-- 使用 wx:for-item 指定当前项，使用wx:for-index 指定索引 -->
    <view wx:for="list" wx:for-item="entry" wx:for-index="no">
        {{ no }} . {{ entry.title }}
    </view>

    <view wx:for="list" wx:key="index">{{ index }} . {{ item.title }}</view>

</view>
```

```javascript
Page({
    data: {
        list: [
            { title: '标题一' }
            { title: '标题二' }
            { title: '标题三' }
            { title: '标题四' }
            { title: '标题五' }           
        ]
    }
})
```


### 事件

事件是视图层（WXML）到逻辑层（JS）的通信方式，在用户界面上操作（点击、触摸、长按）触发事件，逻辑层通过绑定的事件处理函数来响应。


#### 事件的分类

从两个角度对事件进行分类：事件流和组件的角度。

##### 事件流

和浏览器的 DOM 事件流一样，小程序的事件流分为三个阶段：捕获阶段、目标阶段和冒泡阶段。同样，小程序的事件分为冒泡事件和非冒泡事件。



- 常用的冒泡事件列表。
    
    | 事件类型       | 触发条件                 |
    |----------------|--------------------------|
    | tap            | 用户点击                 |
    | longpress      | 用户长按                 |
    | touchstart     | 手指触摸动作开始         |
    | touchmove      | 手指触摸后移动           |
    | touchend       | 手指触摸动作结束         |

- 常见的非冒泡事件。除上述冒泡事件外，其他组件的自定义事件默认都是非冒泡事件。

    | 事件类型       | 触发条件                                 |
    |----------------|-----------------------------------------|
    | `<form>`       | `submit`、`reset`                        |
    | `<input>`      | `focus`、`blur`、`input`、`confirm`       |
    | `<scroll-view>`| `scroll`、`scrolltoupper`、`scrolltolower`|
    | `<swiper>`     | `change`、`transition`                    |


##### 组件事件

微信小程序的组件全都是微信内置的，它和DOM常规的事件是不一样的。所以需要知道哪些组件支持那些事件

- 通用事件。这是所有组件都支持的事件

    | 事件类型       | 触发条件                                 |
    |----------------|-----------------------------------------|
    | tap            | 手指触摸后，超过350ms再离开，触发        |
    | touchstart     | 手指触摸动作开始                         |
    | touchmove      | 手指触摸后移动                           |
    | touchend       | 手指触摸动作结束                         |
    | touchcancel    | 手指触摸动作被打断，如来电提醒，弹窗     |
    | longpress      | 手指触摸后，超过350ms再离开，触发        |

- 组件自定义事件。这类事件是特定组件才会有的，需要查阅组件文档。
 
#### 事件绑定的方式

- bind ： 绑定并允许冒泡
    
    ```xml
    <view id="outer" bindtap="handleTap1">
        outer view
        <view id="inner" bindtap="handleTap2">inner view</view>
    </view>
    ```

    点击 `inner view` 时：
    ```text
        handleTap2 先执行 → 冒泡 → handleTap1 后执行
    ```

- catch ： 绑定并阻止冒泡。

    ```xml
    <view id="outer" bindtap="handleTap1">
        outer view
        <view id="middle" catchtap="handleTap2">
            middle view
            <view id="inner" bindtap="handleTap3">inner view</view>
        </view>
    </view>
    ```
    
    点击 `inner view` 时：
    ```text
    handleTap3 执行 → 冒泡到 middle → 被 catch 阻止 ❌ 不再向上传 → handleTap1 不会执行！
    ```

    点击 `middle view` 时：
    ```text
    只有 handleTap2 执行
    ```

- capture-bind / capture-catch ： 捕获阶段绑定。

    事件传播的两个阶段：
    ```text
    ┌─────────────────────────────────────┐
    │  捕获阶段（从外到内）                │  ← capture-bind 在此阶段触发
    │  outer ──→ middle ──→ inner         │
    ├─────────────────────────────────────┤
    │  冒泡阶段（从内到外）                │  ← bind/catch 在此阶段触发
    │  inner ←── middle ←── outer         │
    └─────────────────────────────────────┘
    ```

    ```xml
    <view id="outer" 
        bind:touchstart="handleTap1"
        capture-bind:touchstart="handleTap2"
    >
        outer view
        <view
            id="inner"
            bind:touchstart="handleTap3"
            capture-bind:touchstart="handleTap4"
        >
            innerView
        </view>
    </view>
    ```

    点击 `inner view` 时的执行顺序：
    ```text
        ① handleTap2 (outer 的捕获)  ← 外到内
        ④ handleTap4 (inner 的捕获)
        ③ handleTap3 (inner 的冒泡)   ← 内到外
        ② handleTap1 (outer 的冒泡)
    ```

    如果将第一个 `capture-bind` 改为 `capture-catch`:
    - 只会执行 `handleTap2` , 然后捕获和冒泡都会终止。

- mut-bind ： 互斥绑定。

    所有 `mut-bind` 之间是互斥的，只会有其中一个被触发，但不影响 `bind` 和 `catch`.
    ```xml
    <view id="outer" mut-bind:tap="handleTap1">
        outer view
        <view id="middle" bindtap="handleTap2">
            middle view
            <view id="inner" mut-bind:tap="handleTap3">innerView</view>
        </view>
    </view>
    ```

    点击 `inner view` 时:
    ```text
    handleTap3 (mut-bind, 最内层先触发) ✅
    handleTap2 (bind, 正常冒泡)          ✅
    handleTap1 (mut-bind, 被互斥)         ❌ 不触发
    ```

    适用场景：多个同级元素需要互斥行为时（如单选按钮组）。

#### 额外数据传递

有时候在处理事件时，需要传递一些额外的数据，有下面几种方式。

- 方式1：`data-*`自定义属性 + dataset（最常用✅）。
    
    ```xml
    <view 
        bindtap="handleTap" 
        data-id="{{id}}"
        data-name="{{name}}"
        data-index="{{index}}"
    >

    </view>
    ```
    ```javascript
    Page({
        data: {
            id: 1,
            name: '张三',
            index: 0

        },
        handleTap(e) {
            // 获取传入的数据
            const { id, name, index } = e.currentTaget.dataset;
        }
    })
    ```

    这里有个需要注意的点，小程序内部会对绑定的属性做转换，所以我们访问这个属性的时候也需要注意：
    
    - 连字符会被转换为驼峰命名法，例如 `data-user-info` 会被转换为 `userInfo`。
    - 大写会被转换为小写，例如 `data-UserInfo` 会被转换为 `userinfo`。
    
    ```xml
    <view
        bindtap="handleTap"
        data-user-info="{{userInfo}}"
        data-userInfo="{{userInfo}}"
    >
    </view>
    ```
    ```js
    Page({
        data: {
            userInfo: {
                name: '张三',
                age: 18
            }
        },
        handleTap(e) {
            e.currentTaget.dataset.userInfo === 1; // 连字符转驼峰
            e.currentTaget.dataset.userinfo === 1; // 大写转小写
        }
    })
    ```

- 方式2：`mark` 标识符（跨层透传 ✅）

    `mark` 的主要作用是识别具体触发事件的 `target` 节点, 但是也可以被用于在组件树的多个层级之间传递数据。 它和 `dataset` 不同，它会合并冒泡路径上的所有 `mark` 数据。

    ```xml
    <view mark:parentMark="parent" bindtap="onParentTap">
        parent
        <button mark:childMark="child" bindtap="onChildTap">按钮</button>
    </view>
    ```

    ```
    Page({
        onChildTap(e) {
            console.log(e.mark.parentMark, e.mark.childMark); 
        }
        onChildTap(e) {
            console.log(e.mark.parentMark, e.mark.childMark); 
        }
    })
    ```


- 方式3：闭包（不推荐）


### 模板

模板的主要作用是实现 UI 结构的复用，模板自身不包含数据及数据处理逻辑。定义和使用模板需要注意：

1. 定义模板时使用 `name` 属性指定模板的名称，使用时通过 `is` 属性指定要使用的模板。
2. 模板的数据通过 `data` 属性传入。

模板的定义和使用有两种方式：

- 方式一： 在当前文件内定义（同文件）。

    ```xml
    <!-- 定义模板 -->
    <template name="myTemplate">
        <view>
            {{item.title}}
        </view>
    </template>

    <!-- 使用模板 -->
    <template is="myTemplate" data="{{item: list[0]}}"></template>
    ```

- 方式二： 在单独的 `wxml` 文件中定义，在跨页面复用模板时使用。
    1. 步骤一：在单独文件中定义模板。
        ```xml
        <!-- templates/item.wxml <独立的模板文件> -->
        <template name="item">
            <view class="item-card">
                <text>{{ text }}</text>
                <image src="{{ imgUrl }}" mode="aspectFill">
            <view>
        </template>
        ```
    2. 步骤二： 在页面中引入并使用模板
        ```xml
        <!-- pages/index/index.wxml -->

        <!-- 第一步： import 引入外部模板文件 -->
        <import src="../../templates/item.wxml">

        <view class="list">
            <!-- 第二步： 使用引入的模板 -->
            <template is="item" data="{{...listItem1}}" />
            <template is="item" data="{{...listItem2}}" />
            <template is="item" data="{{...listItem3}}" />
        </view>
        ```
        ```javascript
        Page({
            data: {
                listItem1: {
                    text: '标题1',
                    imgUrl: '/images/pic1.png'
                },
                listItem2: {
                    text: '标题2',
                    imgUrl: '/images/pic2.png'
                },
                listItem2: {
                    text: '标题3',
                    imgUrl: '/images/pic3.png'
                }
            }
        })
        ```
    
### 引入

WXML 提供两种文件引用方式import和include。


- `import` 不会递归传递，这和 CSS 的 `@import` 不同。
    
    在[模板](#模板)那一节讲了 `import` 的使用方式，这里不再赘述，只说明下作用域。

    ```
    A.wxml  →  定义了模板 A
    ↑
    │ import
    ↓
    B.wxml  →  定义了模板 B，同时 import 了 A（B 可以用 A）
    ↑
    │ import
    ↓
    C.wxml  →  只 import 了 B
            ✅ C 可以用 B 的模板
            ❌ C 不能用 A 的模板！（必须显式 import A）
    ```

    |特性      | `import`         | `include` |
    |----------|------------------|---|
    | 引入方式 | 同步 | 异步 |
    | 传递性 | 否 | 是 |
    | 可用性 | 页面内 | 页面间 |
    | 使用方式 | 在页面中使用 | 在组件中使用 |
    | 使用限制 | 不支持循环引入 | 不支持循环引入 |
    | 使用场景 | 页面内复用 | 页面间复用 |
    | 使用示例 | `<import src="..." />` | `<include src="..." />` |

- `include` 可以将目标文件除了 `<template/>` 和 `<wxs/>` 以外的整个代码引入：

    ```xml
    <!-- header.wxml -->
    <view>header</view>

    <!-- footer.wxml -->
    <view>footer</view>

    <!-- index.wxml -->
    <include src="footer.wxml" />
    <view>body<view>
    <include src="footer.wxml" />
    ```
