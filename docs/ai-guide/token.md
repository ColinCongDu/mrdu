# token

`token` 是大模型读写文本时的最小计算单位。大模型是无法直接理解“苹果”或者“apple”这种抽象概念的，模型的本质是数学函数，它处理文本的过程大概如下：它先把文本拆成一小块一小块的单位（`token`），并将其映射为数字（`ID`），然后将这些数字转换为向量（`vector`），最后输入到神经网络中进行计算。
```
文本 → Token → 数字（ID）→ 向量 → 神经网络 -> Token -> 文本
```
`token` 的切分规则（Tokenization）并不是固定的，它取决于模型使用的分词器（Tokenizer）。
- 英文规则：通常是一个单词，或者长单词的一部分。例如 Learning 可能是 1 个 `token`，而 Transformer 可能被拆分为 Trans 和 former。
- 中文规则：由于中文没有空格，分词更复杂。通常一个汉字对应 1-2 个 `token`，词组（如“人工智能”）有时会被视为整体。
- 符号与空格：标点符号和代码中的缩进（Tab/Space）也都是独立的 Token。

模型的计算复杂度，跟 `token` 数量有关, token 越多，计算量是平方级增长。Transformer 的注意力机制，本质上是在 `token` 之间两两建立关系，所以模型限制的是 `token` 的数量（而非大小，例如多少KB），并且按 `token` 数量计费。

**还有一个问题：为什么大模型无法直接理解一整句话，把整句话都映射为数字不可以吗？**

这个问题直戳大模型（LLM）底层设计的核心。作为一个 AI 专家，我可以用一个形象的比喻和三个硬核的技术理由来帮你拆解：**为什么 AI 不能把“整句话”当成一个数字。**

##### 1. 组合爆炸问题（数学上的不可能）

如果把“一整句话”映射为一个数字（Token），那么 AI 的“字典”会瞬间爆炸。

* **单词量**：英语常用单词约 10 万个。
* **句子量**：一句话如果只有 10 个词，可能的排列组合就是 $100,000^{10}$。这个数字比宇宙中的原子数量还要多。
* **后果**：AI 永远无法看完所有可能的句子。如果有一句话 AI 没见过（哪怕只改了一个逗号），它就无法对应到任何数字，彻底“宕机”。

**Token 的逻辑是**：我只需要记住几万块“积木”（Token），就可以通过不同的组合，拼出无限种句子。**这叫“生成能力”，而不是“查表能力”。**

---

##### 2. 泛化能力（AI 必须能“举一反三”）

如果 AI 把一整句话看作一个孤立的数字，它就失去了理解**内部结构**的能力。

* **句子 A**：“猫在写代码。”（数字 ID：1001）
* **句子 B**：“狗在写代码。”（数字 ID：9999）
* 如果它们是两个独立的数字，AI 无法从数学上发现这两句话其实只有一个词的区别。
* **Token 的逻辑是**：AI 看到 `猫` + `在` + `写代码`。当它下次看到 `狗` 时，它能利用之前学习到的 `在写代码` 的规律，立刻理解“狗在写代码”是什么意思。

##### 3. “预测下一个字”的任务本质

目前主流的大模型（Transformer 架构）本质上都是**概率预测引擎**。

AI 的工作流程是：

1. 输入一串数字（Token 序列）。
2. 计算：根据前面的数字，下一个数字最可能是谁？
3. 输出：预测出的那个数字。

**如果一整句话就是一个数字，那么 AI 就无法在句子内部进行推理。** 它只能预测“下一句话”，这会导致生成的颗粒度太粗，根本无法进行细腻的对话或严谨的代码编写。
